🔗 API 端點對應

GET /api/CodeType 取得所有代碼類別
GET /api/CodeType/{id} 根據ID取得代碼類別
POST /api/CodeType 建立新代碼類別
PUT /api/CodeType/{id} 更新代碼類別
DELETE /api/CodeType/{id} 刪除代碼類別
GET /api/CodeType/exists/{codeType} 檢查代碼是否存在

🧪 測試指南

1 啟動開發伺服器
進入前端專案目錄，執行 npm run dev 啟動開發伺服器
2 登入系統
訪問 http://localhost:3000，使用有效的帳號登入系統
3 進入代碼類別維護
在側邊選單中點擊「系統設定」展開，然後選擇「代碼類別維護」
4 測試基本功能
• 查看代碼類別列表
• 使用搜尋功能過濾資料
• 測試分頁功能
5 測試新增功能
• 點擊「新增代碼類別」按鈕
• 填寫必填欄位
• 測試表單驗證
• 確認資料儲存
6 測試編輯功能
• 點擊任一記錄的編輯按鈕
• 修改資料並儲存
• 確認更新成功
7 測試刪除功能
• 點擊刪除按鈕
• 確認刪除對話框出現
• 測試關聯性檢查（如果有相關系統代碼）
8 測試響應式設計
• 調整瀏覽器視窗大小
• 確認手機版本顯示正常
• 測試觸控操作
